## Project Overview

**DecentraliTrack** ‚Äî A blockchain-backed public infrastructure transparency platform. Citizens, government officials, contractors, auditors, and inspectors manage projects and milestones backed by Polygon Amoy smart contracts, with IPFS document storage and real-time WebSocket updates.

## Development Commands

```bash
# Install all workspace dependencies
pnpm install

# Start full dev stack (frontend + backend, auto-selects free ports)
pnpm dev

# Build all packages with typechecking
pnpm build

# Typecheck all packages
pnpm typecheck

# Typecheck shared libraries only
pnpm typecheck:libs
```

**Per-package commands (use `--filter`):**
```bash
# Frontend
pnpm --filter ./frontend run dev
pnpm --filter ./frontend run build
pnpm --filter ./frontend run typecheck

# Backend
pnpm --filter ./Backend run dev
pnpm --filter ./Backend run typecheck

# Database schema push
pnpm --filter ./lib/db run push
pnpm --filter ./lib/db run push-force

# Smart contracts
pnpm --filter ./contracts run compile
pnpm --filter ./contracts run test
pnpm --filter ./contracts run deploy:amoy

# Regenerate API client + Zod types from OpenAPI spec
pnpm --filter ./lib/api-spec run codegen
```

**Default local URLs:**
- Frontend: http://127.0.0.1:5173
- Backend API + WebSocket: http://127.0.0.1:3001

Override via `FRONTEND_PORT` and `API_PORT` env vars.

### Code Generation Flow

`lib/api-spec/openapi.yaml` ‚Üí Orval codegen ‚Üí `lib/api-client-react/src/generated/` and `lib/api-zod/src/generated/`

Always edit the OpenAPI spec first, then run `pnpm --filter ./lib/api-spec run codegen`. Never edit generated files.

### Backend Architecture (`Backend/src/`)

- `index.ts` / `app.ts` ‚Äî Server entry, Express setup
- `data.ts` ‚Äî In-memory data store (used when `DATABASE_URL` is not set; demo mode)
- `routes/` ‚Äî REST endpoints: auth, projects, milestones, tenders, analytics, blockchain, health
- `services/` ‚Äî Business logic: `blockchainListener.ts` (event polling), `contractService.ts` (write transactions), `anomalyEngine.ts` (10-min interval scanner), `ipfsService.ts` (Pinata)
- `socket/` ‚Äî WebSocket pub/sub for real-time frontend updates
- `middleware/auth.ts` ‚Äî JWT verification middleware

**Demo mode:** The app runs fully without `DATABASE_URL` or blockchain RPC ‚Äî the in-memory store in `data.ts` is used as fallback.

### Frontend Architecture (`frontend/src/`)

- `App.tsx` ‚Äî Root: routing (Wouter), auth context, React Query client, toast
- `pages/` ‚Äî One page per role: `citizen.tsx`, `official.tsx`, `contractor.tsx`, `auditor.tsx`, `inspector.tsx`, `admin.tsx`, `project-detail.tsx`
- `lib/auth.tsx` ‚Äî Auth context: MetaMask wallet flow + demo-mode fallback; JWT + user stored in `localStorage`
- `hooks/use-live-updates.ts` ‚Äî WebSocket subscription for real-time project/milestone events
- `components/ui/` ‚Äî shadcn/ui components (Radix + Tailwind); do not modify generated components
- `abis/` ‚Äî TypeChain-generated contract ABIs; regenerated by `contracts` compile step

### Smart Contracts (`contracts/contracts/`)

Deployed on **Polygon Amoy** (chainId 80002):
- `RoleManager.sol` ‚Äî On-chain role assignments: `GOVERNMENT_OFFICIAL`, `AUDITOR`, `CONTRACTOR`, `INSPECTOR`, `CITIZEN`
- `ProjectRegistry.sol` ‚Äî Project lifecycle: create ‚Üí approve/reject ‚Üí close
- `MilestoneEscrow.sol` ‚Äî Fund escrow with proof-based milestone release

Contract addresses are provided via environment variables (`ROLE_MANAGER_ADDRESS`, `PROJECT_REGISTRY_ADDRESS`, `MILESTONE_ESCROW_ADDRESS`).

### Data Flow

1. MetaMask signature ‚Üí Backend issues JWT
2. Frontend calls REST API (Bearer token) ‚Üí Backend may call contracts via ethers.js
3. `blockchainListener.ts` polls contract events ‚Üí broadcasts over WebSocket
4. Frontend `use-live-updates` hook receives events ‚Üí React Query cache invalidated ‚Üí UI updates

## Environment Variables

Copy `.env.example` to `.env`. Key variables:

| Variable | Purpose |
|---|---|
| `DATABASE_URL` | PostgreSQL connection string (optional; demo mode if absent) |
| `SESSION_SECRET` | JWT signing secret |
| `POLYGON_AMOY_RPC` | Blockchain RPC endpoint |
| `DEPLOYER_PRIVATE_KEY` | Contract deployer wallet |
| `PINATA_JWT` / `PINATA_GATEWAY` | IPFS via Pinata |
| `VITE_*` | Frontend-only vars (contract addresses, gateway URLs) |

## Key Technology Choices

- **pnpm workspaces** ‚Äî enforces `pnpm` only (preinstall check); `npm`/`yarn` will fail
- **TypeScript project references** ‚Äî incremental compilation; `tsconfig.base.json` is the shared base
- **Drizzle ORM** ‚Äî schema in `lib/db/src/schema/index.ts`; run `push` (not migrations) to sync
- **Orval** ‚Äî regenerates both React Query hooks and Zod validators from a single OpenAPI file
- **esbuild** ‚Äî Backend bundles to `Backend/dist/index.mjs` (ESM); native modules are externalized
- **Pino** ‚Äî structured JSON logging on the backend; use `req.log` inside Express handlers
- **shadcn/ui** ‚Äî component config in `frontend/components.json`; add new components via the shadcn CLI
